** JVM

	* "Write once, Run anywhere". 


	* "WORA"


	* "Escreva uma vez, execute em qualquer lugar".


	* “roda em qualquer lugar”


	* Java Virtual Machine, Maquina Virtual Java


	* É um dos principais conceitos para entender a própria tecnologia
Java


	* Bytecode: é um código intermediario, gerado quando compilamos
o codigo fonte java


	* Mostrar Hello World em Bytecode


	* E esse Bytecode (.class) que vai ser usado pela JVM


	* Freezar que a JVM não entende Java, e sim Bytecode


	* Pensar no trajeto Java > Compiler > JVM


	* O que é uma maquina virtual?


	* É a simulação de um computador


	* Gerenciar processo, memoria, arquivos e etc, mas não possui
hardware fisico


	* Voltando: Maquina virtual do Java


	* Ela é responsavel por traduzir o codigo para o sitema operacional
ou o dispositivo que tem a JVM instalado


	* Ou seja precisa ter a JVM instalado


	* Não é um simples compilador (não confundir)


	* Compila e interpreta ao mesmo tempo, gerencia pilhas e threads,
como a Garbage collector que automaticamente limpa a memoria de coisas que
não estão sendo usadas


	* Complexidade:


	* Outras linguagem: trajeto de codigo fonte > compilador > sistema
operacional


	* Porque cada sistema ou dispositivo funciona de forma diferente
(Mac, Windows, Linux)


	* Então precisa reescrever o código para cada sistema, apontando
para bibliotecas e recursos daquela sistema e usar um compilador diferente


	* Mostrar compiladores


	* A JVM veio para contornar esse problema


	* Mostrar o compilador java


	* A JVM converte o Bytecode para o codigo de maquina daquela
maquina especifica


	* O dispositivo, sendo uma TV, um celular ou até um chip, tem a
JVM instalada e é obrigação dela interpretar o Bytecode


	* JVM é feita em código nativo daquele sistema


	* Cada dispositivo tem uma JVM dele, Windows vs Linux


	* Por que não programar diretamente em Bytecode?


	* O Objetivo da linguagem é ser simples, a JVM depende de um codigo
mais complexo para ser ampla em todas as plataformas, por isso 2 conversões


	* Citar o CLR
