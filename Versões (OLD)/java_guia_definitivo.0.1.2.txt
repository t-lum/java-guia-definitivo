↪ O que é uma Linguagem de programação?

	Um bom jeito de se imaginar o que é uma linguagem de programação é pensar em um
idioma. É uma forma de conseguir se comunicar com o computador. Possuimos uma
nacionalidade que na maioria dos casos determina o nosso idioma, seja português, inglês,
francês ou qualquer outro no mundo. O mesmo se aplica a computadores ou dispositivos
eletronicos, eles podem compreender ordens e rotinas através de sua "linguagem", o sistema
binário (0 e 1).


	Programar não é necessariamente escrever em 0 e 1, é uma ativade em etapas: primeiro
escrevemos o código fonte, que posteriormente um compilador/intepretador converte para
código de maquina. Nessa ocasião o compilador se comporta como um tradutor, conseguindo
traduzir a nossa mensagem para maquina sem que perca o seu sentido.


	Esse código fonte é escrito em alguma linguagem de programação, podendo ser C#,
Java, Python, C++ e etc (uma forma boa de se compreender é imaginar de novo, português,
inglês ou francês). A única diferença entre uma linguagem outra é quase a mesma que a
diferença entre os idiomas: possuem regras e sintaxes diferentes. Porém as diferenças nas
linguagens podem ter impacto nas ferramentas que oferecem.


	Entre as especialidades e características das linguagens, destace-se os seus niveis.
Antes de tudo, esses níveis não são sobre o grau de dificuldade em seu aprendizado
espeficamente, eles são sobre o nível de compreensão humana. Quanto maior o nivel da
linguagem mais próxima ela é dos humanos, enquanto mais distante das maquinas. O contrario
também se aplica:


	* Linguagem de alto nivel: compreensão mais proxima de humanos


	* Linguagem de baixo nivel: compreensão mais proxima das maquinas


	Esses níveis na pratica resumem a abstração da linguagem e a complexidade em ser
copiladas / interpretadas para a maquina (tradução). É importante avisar que não é uma regra
uma linguagem de baixo nivel ser impossivel de se aprender ao mesmo tempo que uma de
alto nivel tenha pouco controle do hardware da maquina, as linguagens de programação também
são um produto comercial e tentam evoluir para continuar ativas na competição entre qual os
desenvolvedores vão escolher.


	* Nota: Java é considerada uma linguagem de alto nível.



↪ O que é um Algoritmo?

	Com um pouco de compreensão sobre linguagens de programação é mais facil entender o
o que é um algoritmo. Por definição é uma rotina, uma sequencia finita ou infinita de
passos:


	* Exemplo:

{

	SE LÂMPADA APAGADA

        	ENTÃO
	   		ACENDER LÂMPADA

		SENÃO
			NÃO FAZER NADA
    	FIM

}


	Existem algumas ferramentas para auxiliar novos desenvolvedores, entre elas temos o
Portugol, um pseudocódigo feito para simular programação através de palavras em português
para facil entendimento. Portugol não é uma linguagem de programação, apenas uma 
pseudolinguagem (pseudo = superficialmente).


	* Hello World em Portugol:

{
	algoritmo "Olá mundo"
 
	var
 
	inicio

	escreva ("Olá mundo!")
 
	fimalgoritmo
}


	* Hello World em Java:

{

	class Program {
		public static void main(String args[]){
			System.out.println("Hello World");
		}
	}

}


	* Obs: não há necessidade de entender o que cada linha do código significa, é apenas
um exemplo do mesmo algoritmo escrito em uma pseudolinguagem (Portugol) e o outro em uma
linguagem de programação (Java).


	* Obs: Ambos códigos quando executados retornam "Hello World" no console (prompt).


	O algoritmo é como uma receita de bolo, lista de compras a se fazer, qualquer
conjutos de ações é um algoritmo. Somando a linguagens de programação, o algoritmo pode ser
escrito em diversas linguagens e podendo ter o mesmo sentido.

//
//	//	JVM
//


   "Write once, Run anywhere", escreva uma vez, rode em qualquer lugar.


   O que seria a JVM? JVM (Java Virtual Machine) é maquina virtual do java, o responsavel
pelo bordão acima. O objetivo dos criadores do Java não era criar mais uma linguagem de
programação, mas uma integração entra varios sistemas operacionais diferentes, como
dispositivos e equipamentos. Então temos essa solução, a JVM, uma máquina virtual que pode
converter os Bytecodes java para código de maquina executável daquele dispositivo (se tiver
uma versão de JVM é claro).


   Exemplo para entender melhor: imagine que cada sistema operacional ou dispositivo seja um
represetante de um país diferente. O programador quer usar um discurso, exatamente o mesmo, para
cada representante com a intenção de que eles entendam, para isso existe um "tradutor" que vai
"traduzir" o que você disse para cada um deles.


   Essa é lógica da JVM, uma maquina virtual que transforma Bytecodes em código daquela maquina


   A ordem de passos feita pelo compilador:


1) Codigo Fonte → Feito pelo programador

2) Bytecode → codigo intermediario

3) Maquina Virutal → JVM no caso de Java

4) Sistema Operacional

5) Hardware → maquina fisica


   * Bytecode: O que é escrito no código fonte (feito pelo programador) é compilado para uma
forma intermediaria, chamada Bytecode, que é interpretada pelas máquinas virtuais (JVMs).


   * Compilador: é uma aplicação que converte código fonte para linguagem compilada, traduzindo
uma linguagem mais compreensivel a humanos para uma linguagem mais compreensivel a maquinas.


   Observação: é muito dificil escrever isso sem citar o CLR, que é praticamente a máquina
virutal da .NET (C#, seu "concorrente"). Praticamente a mesma logica, todos os dispositivos tem
uma versão de .NET, códigos C#, VB ou outro código .NET é compilado para CLR (Common Language
Runtime) que é traduzido para código de maquina para esses dispositivos.

//
//	//	Compilação vs Interpretação
//


   Além do que foi explicado de linguagem intermediaria (compilação) existe a interpretação,
algums linguagens tem o seu design para ser comumente interpretadas ao inves de compiladas.


   Qual a diferença? A compilação gera um arquivo de linguagem intermediaria enquanto o
interpretador precisa apenas interpretar o que ja foi escrito no codigo fonte. Olhando desse
ponto um parece melhor que o outro mas as 2 formas possuem vantagens e desvantagens.


↪ Compilação

  ↪ O que é: a leitura e tradução do codigo fonte para gerar um codigo intermediario a ser
executado por uma maquina virtual.


  ↪ Vantagens:

    ↪ velocidade da aplicação, já que não precisa ficar interpretando toda vez que é executado,
é só utilizar o codigo intermediario já traduzido.


    ↪ Auxilio do compilador, como o codigo é todo lido para ser traduzido, já é possivel
indentificar erros antes de sua execução.


↪ Intepretação

  ↪ O que é: o código é interpretado sem a necessidade de gerar um arquivo de codigo
intermediario.


  ↪ Vantagens:

    ↪ flexibilidade de manutenção, já que possui menos arquivos no total e precise de apenas
modificar ao inves de modificar e compilar para gerar o arquivo compilado modificado.


    ↪ Recompilação, não precisa ser recompilado para cada plataforma que o consegue interpretar,
já que é póssivel a sua leitura.

  O que é IDE?


   IDE (Ambiente de Desenvolvimento Integrado, Integrated Development Environment) é uma aplicação
que reune varias ferramentas e caracteristicas para ajudar no desenvolvimento de outras aplicações.


  Resumindo: um software para fazer software.


   IDE não é nada mais do que um aplicativo com principalmente um editor de texto e um compilador,
mais algumas ferramentas para facilitar o trabalho do programador. Existe discussões sobre usar ou
não, a esmagante maioria usa, mas é sempre importante entender o não uso e o uso já que é
facilitado demais e as vezes faz esquecer como as coisas realmente funcionam.


   Sem IDE o programdor tem que usar um editor de texto (até bloco de notas), compilar pelo 
CMD/Terminal e gerenciar seus arquivos em uma pasta com sua propria organização, o que não é muito
agradavel a novos desenvolvedores.

//
//	//	JDK vs JRE
//


   Antes de instalar temos que entender o historico e versões do java.


   Na verdade é bem simples:



↪ JDK

  ↪ Java Development Kit, Kit de Desenvolvimento Java


  ↪ É um pacote de ferramentas, sendo necessario para o desenvolvimento de aplicativos java


↪ JRE

  ↪ Java Runtime Environment, Ambiente de Tempo de Execução Java


  ↪ É um plug-in necessario para poder executar programas Java, sendo menor que a JDK



   Resumindo JRE é o minimo suficiente para que um usuario-cliente possa executar aplicações java,
aqueles que eu e você já baixamos uma vez para executar um game ou software. Enquanto JDK é o
pacote completo com uma series de ferramentas.


   JRE possui a JVM, que já expliquei, e as bibliotecas de classes.


   JDK possui JRE, tanto JVM quanto as bibliotecas, quanto outras ferramentas como compiladores,
debuggers, documentações entre outros.

atalho na area de trabalho, essa é bem opcional)
